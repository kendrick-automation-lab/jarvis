기간

성과

내용
  • 배경 · 특정 회사를 정하지 않고 중소 제조·유통사에서 흔한 반복 업무를 골랐습니다. 실제 ERP(이카운트)와 협업툴(Flow)은 데모 계정으로는 API가 막혀, Airtable과 Slack으로 같은 구조를 짰습니다
  • 구현 · 안전재고 확인이나 입금 대조처럼 규칙이 분명한 일은 코드로 처리하고 열린 질문만 에이전트로 넘겼습니다. 답이 조회된 데이터 안에서만 나오도록 제한하고, 확인되지 않으면 모른다고 답하게 했습니다. 발주나 입금처럼 결정이 필요한 일은 사람이 승인합니다
  • 결과 · 이전 대화의 거래처와 맥락을 이어가는 조회를 넣었습니다. 재고·재무·미수 분석을 병렬로 돌려 정해진 시각에 팀 채널로 보내는 멀티에이전트 브리핑까지 붙였습니다. 합성 데이터로 만든 대시보드와 Slack 봇으로 테스트했습니다

기간
2026.05 ~ 2026.06
성과

내용
  • 배경 · 조작 가능한 API가 없어 화면을 실시간으로 캡처해 픽셀 단위로 상태를 판정해야 했습니다. PC와 노트북은 해상도가 서로 달라, 환경마다 좌표가 어긋나는 문제가 있었습니다
  • 구현 · 템플릿 매칭과 HSV 색상 기반 BLOB 검출을 병행했습니다. 게임 창을 1298×767로 표준화하고 절대 좌표를 0~1 상대 좌표로 정규화해, 해상도와 무관하게 동일한 좌표 로직을 적용했습니다. 환경별 값은 설정 파일로 분리했습니다
  • 결과와 개선 · 운영 중 발생한 입력 누락·포커스 이탈·오탐지를 안정화 패치 8회로 해결했습니다. 이동 가능 경계가 좌우로 비대칭(좌 0.04·우 0.77)인 것을 실측해 반영했습니다. 18일간 커밋 267개, 약 1만 줄 규모로 개발했습니다

기간
2026.07 ~ 현재 (서버 24시간 운영)
성과
9개 체인과 여러 거래소에서 650여 개 종목을 실시간으로 지켜보며, 가격 이상이나 대량 자금 이동, 지갑 집중도 같은 신호 10여 종을 통합 감지하는 시스템입니다. 개인 연구용으로 서버에서 24시간 운영하고 있습니다.
내용
  • 배경 · 이상 신호는 여러 거래소와 체인에 잠깐씩 나타났다 사라집니다. 눈으로 다 좇기 힘들어 자동으로 지켜보게 했습니다
  • 구현 · WebSocket으로 종목을 실시간 수집하고 신호를 종류별로 나눠 감지합니다. 여러 신호가 한 종목에 겹치면 더 확실한 신호로 보고 알립니다. 온체인 쪽은 9개 체인의 지갑과 입금 흐름까지 확인합니다
  • 검증과 한계 · 감지한 신호는 실제 가격 결과로 채점해, 성적이 나쁜 신호는 자동으로 비중을 낮춥니다. 다만 수익성이 검증됐다고 보지는 않고, 어떤 신호가 실제로 맞는지 데이터로 쌓아 가는 단계입니다. 배포 뒤 생긴 WebSocket 끊김은 비동기 구조로 바꿔 해결했습니다

성과

내용
  • 배경 · 밈코인 시장은 새 토큰이 끊임없이 쏟아집니다. "떴다"만으로는 부족하고 "왜 떴는지"까지 알아야 판단할 수 있었습니다

내용

핵심 역량

  • AI(Claude Code·Codex)로 필요한 자동화를 직접 만들어 왔습니다 (6개월간 19개)
