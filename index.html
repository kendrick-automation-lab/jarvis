01주요 자동화 프로젝트

사람이 다 볼 수 없는 일을
대신 보고 처리하게 만들었습니다.

공통적으로 "사람이 실시간으로 전부 확인·판단하기 어려운 일"에서 병목을 찾아, 규칙이 분명한 건 코드로, 판단이 필요한 건 AI로 나눠 자동화했습니다.

[개인 프로젝트]Claude 기반 자율 의사결정 에이전트
성과
사람이 다 보고 판단할 수 없는 수십 개 시장을 Claude가 분석·판단하고, 검증과 안전장치를 거쳐 자동 실행하게 만들었습니다. AI를 '의사결정 두뇌'로 쓰되 그대로 믿지 않고 가드레일로 감싸는 구조를 직접 설계했습니다.
내용
  • 왜 · 대상이 너무 많아 사람이 일일이 분석·판단할 수 없었습니다. AI에 판단을 맡기되 실행은 통제해야 했습니다
  • 어떻게 · 30분마다 후보 약 100개를 상위 20개로 필터 → Claude API로 판단을 JSON으로 받음 → 유효성·예산 한도 검증으로 잘못된 결정 거부 → 모의/실행 분기 → 텔레그램 보고
  • 결과 · 모의 모드로 사이클을 안전하게 검증하며 운영, 판단·결과를 로그로 남겨 다시 확인 가능
  • 개선 · 하나하나 열어 보고 판단하던 걸 AI가 자동으로 훑어 상위만 추려 판단 → 사람이 안 붙어 있어도 돌아가게. AI를 그냥 믿지 않고 검증·한도로 감싼 설계
[개인 프로젝트]실시간 거래소·온체인 이상거래 탐지 시스템
성과
여러 거래소와 블록체인에서 이상거래·시장 조작 신호를 실시간으로 감지하는 시스템을 만들어 서버에서 24시간 운영하고 있습니다. 감지한 신호를 실제 결과로 채점해, 성과가 낮은 신호는 자동으로 걸러지도록 만들었습니다.
내용
  • 왜 · 이상 신호가 여러 거래소·체인·토큰에 짧게 흩어져 발생해, 사람이 실시간으로 전수 감시하기 어려웠습니다
  • 어떻게 · WebSocket으로 650여 개 종목을 실시간 수집 → 가격 이상·대량 자금 이동·지갑 집중도 등 10여 종 신호를 통합 감지 → 여러 신호가 겹치면 확증으로 승격 → 텔레그램 알림
  • 결과 · 서버에서 24시간 안정 운영 (초기의 WebSocket 잦은 끊김을 비동기 구조로 근본 해결)
  • 개선 · 신호를 그냥 믿지 않고 실제 가격 결과로 승률·기대값을 채점해, 검증된 신호만 알림을 보내고 성과가 나쁜 신호는 자동 차단 → "맞는 신호만 남는" 자가교정 구조
[개인 프로젝트]회사 업무 자동화 PoC — 사내 조회 에이전트
성과
회사의 사내 업무를 가정해, 자연어로 물으면 데이터를 조회해 답하는 에이전트와 반복 업무 자동화(안전재고 알림·입금 매칭·전표 초안)를 만들었습니다. 정답을 정한 평가 5건으로 검증해 모두 통과했습니다.
내용
  • 왜 · 회사엔 사람이 반복하는 조회·확인·대조 업무가 많습니다. 규칙이 분명한 일과 판단이 필요한 일을 나눠 자동화하고 싶었습니다
  • 어떻게 · 안전재고 확인·입금 대조처럼 규칙이 분명한 일은 코드로, 자유롭게 들어오는 질문은 에이전트가 조회 도구로 처리. 실제 ERP·협업툴은 데모 제한이 있어 Airtable·Slack에 같은 구조를 구현
  • 결과 · 합성 데이터로 대화형 조회 · 반복 업무 자동화 · 여러 분석을 모으는 아침 브리핑까지 동작 확인
  • 개선 · AI가 없는 내용을 지어내지 못하게 조회 전용 도구만 주고, 발주·입금 같은 결정은 사람이 승인하도록 범위를 나눔 (AI가 맡을 일과 사람이 맡을 일을 구조로 분리)
[개인 프로젝트]화면 인식 기반 게임 자동화
성과
조작할 API가 없는 환경에서 반복 조작을 '화면을 보고' 자동 처리하게 만들었습니다. 캡처→인식→조작 파이프라인과, 기기마다 다른 환경(해상도·DPI)을 견디는 설정 구조를 직접 설계했습니다.
내용
  • 왜 · 반복 조작이 필요한데 조작할 API가 없어, 화면을 직접 읽어 상황에 맞게 움직이는 자동화가 필요했습니다
  • 어떻게 · 화면 캡처 → OpenCV 템플릿 매칭으로 요소 인식(정확도 0.65~0.85) → 좌표 기반 자동 조작 + 위험 감지 시 즉시 정지. 설정 계층을 분리해 PC·노트북 2대에서 같은 코드로 운영
  • 결과 · 2대 기기에서 장시간 무인 운영
  • 개선 · 똑같은 반복 조작에 들던 사람 손을 0으로. 창 좌표·인식 기준값을 실측으로 잡아 개입 없이 유지되게 안정화
[게임잼 출품]메이플스토리 IP 2D 서바이버 게임
성과
넥슨 메이플스토리 공식 IP 게임잼에 2D 서바이버 게임을 만들어 출품·배포했습니다. 이동·자동공격·레벨업 성장·보스·리더보드까지 갖춘 한 판 완결형 게임을 구현했습니다.
내용
  • 왜 · 공식 IP 애셋과 플랫폼이 주어진 게임잼에서, 짧은 시간에 "30초 안에 빠져들고 한 판 더 하게" 되는 완결형 게임을 목표로
  • 어떻게 · 자동공격 투사체·군집 스폰·레벨업 랜덤 3택1·보스·영구강화 상점을 구현. 게임 엔진(Phaser)으로 렌더하고 게임서버로 점수 리더보드를 연동
  • 결과 · 실제 플랫폼에 배포해 플레이 가능한 상태로 제출
  • 개선 · 여러 차례 플레이테스트로 조작감·밸런스·UI를 다듬어, 붙잡고 한 판 더 하게 되는 흐름으로 완성
[개인 프로젝트]델타뉴트럴 트레이딩 봇
성과
24시간 떠 있는 차익 기회를 사람이 계속 볼 수 없으니 봇이 대신 감시·실행·헤지하게 만들었고, 원하는 대로 돌 때까지 끈질기게 튜닝해 안정화했습니다.
내용
  • 왜 · 차익 기회는 짧고 불규칙해 수동으로는 놓치기 쉬워, 자동 감시·실행·헤지가 필요했습니다
  • 어떻게 · 두 거래소 API(REST·WebSocket)로 실시간 호가를 추적 → 주문·체결을 감지 → 체결되면 반대 거래소에서 다단계로 헤지 (asyncio, 약 1,700줄 엔진)
  • 결과 · 설정값을 백 번 넘게 바꿔가며 안정화, 약 200개 시장에서 운영
  • 개선 · 몇 초~분이면 사라지는 기회를 수동으론 못 잡으니, 두 거래소를 실시간으로 묶어 자동 진입·헤지로 사람 없이 처리
[개인 프로젝트]인사이더·이상거래 감지 봇
성과
실시간으로 쏟아지는 모든 거래를 사람이 볼 수 없으니, 이상·인사이더 패턴을 자동 감지·점수화해 등급별로 알림하게 만들었습니다. 실제 인사이더 사례 5건에서 공통 패턴을 역으로 뽑아 신호로 만들었습니다.
내용
  • 왜 · 모든 거래를 수동으로 감시할 수 없어, 자동 수집·이상 감지가 필요했습니다
  • 어떻게 · WebSocket으로 실시간 거래 수집 → 4개 신호(지갑 신선도·베팅 집중도·시장 맥락·지갑 군집)로 0~100 점수화 → 등급별 알림
  • 결과 · 서버에서 감지·알림, 누적 약 34,446건 분석
  • 개선 · 전체 거래를 다 볼 수 없는데, 실시간 전수 스캔 + 0~100 자동 점수화로 '볼 가치 있는 것'만 골라 올려줌

이 외에도 알림 봇, 데이터 수집·자동화 스크립트 등 여러 프로젝트를 만들었고, 일부는 서버에서 24시간 운영 중입니다.

성과

내용

  • 메일 열람·링크 클릭·설문 응답에 따라 확인 안내·참여 독려·후속 메일이 이어지도록 조건을 나눔

[크립토 마케팅 프리랜서]리서치 · 콘텐츠 · 커뮤니티
성과

할 수 있는 것

  • Claude Code·Codex 기반 작업 흐름으로 약 6개월간 자동화 프로젝트 19개 제작
  • LLM이 질문에 맞는 조회 도구를 스스로 선택하고 결과를 이어 처리하는 tool-use 에이전트 구현
  • "어디에 코드를 쓰고 어디에 AI를 쓸지" 병목을 보고 판단하는 설계
  • WebSocket으로 수백 개 종목을 실시간 수집·처리 (asyncio 비동기)
  • REST·WebSocket·SDK 방식으로 외부 API 10여 종 연동
  • OpenCV 기반 화면 인식과 좌표 자동화
  • Python을 주로 쓰고 JavaScript·PowerShell을 보조로 활용

배우고 있는 것

  • LangChain·CrewAI 같은 정식 오케스트레이션 프레임워크를 실무 프로젝트에 적용해 본 경험은 아직 없습니다
  • 회사에서 실제로 쓰는 ERP·협업툴의 운영 환경은 처음입니다
  • 외부 사용자가 있는 서비스의 팀 개발·운영 경험은 없습니다
